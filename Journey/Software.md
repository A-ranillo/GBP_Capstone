## ROUTER/FIREWALL/NIDS
We will be using opnsense. Why opnsense? because it's the bleeding edge and when doing something that is very very ambitious like this capstone- getting frequent updates and features is very handy. Initially it was supposed to be pfsense BUT I've heard from people pfsense just lacks the support of new features so opnsense it is.

For containerization we used docker for ease of use. To manage the docker containers we have portainer. All docker containers are in docker-compose/docker stack.
